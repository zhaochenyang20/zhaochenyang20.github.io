PA2-2 KiDD题解

一、算法分析

模拟一棵线段树

工欲善其事必先利其器。kidd一题我将大量的时间用于设计和构思，而花费在debug上的时间显著的少。算是一次妙趣横生的编程体验。

本题考察的显然为线段树数据结构，然而考虑到给出的卡牌个数最大为\(2^{31}\)，如果建立一棵叶子节点规模如此巨大的线段树，无疑会超出内存限制，故而需要考虑将叶节点离散化。

具体而言，考虑到操作数最多为200_000，涉及到的端点最多为400_000个，这些端点(排序去重之后)假设为k个。能够将初始的卡牌拆分为k+1个区间，从而将这些区间进行整合，整合为新的叶节点，不妨称之为超节点。

对于每个超节点，似乎仅仅需要记录超节点的长度即可。然而，如此简单操作，会导致在边界出现极为复杂的特判情况非常糟糕。譬如，倘若去重排序后连续的三个节点为20、30、49如果我们先翻动20、30，不论是将增量11放在哪个端点上，都很难处理。

转换一种思路，假设端点排序去重后为数组a，倘若数组a为20,30,33，假想有一个假想的b数组，其实际的含义为区间[20,20],[21,29],[30,30],[31,32],[33]。也即我们加上虚拟的节点来表示原来a数组里的区间，而原本a数组里的值全部为单个点。这样以来，我们可以将每次操作都记录在区间节点上，不至于发生矛盾。对这一假想的区间数组构造线段树。

函数功能设计

二分搜索

对于某次操作的端点m、n，首先在a数组中二分查找出两端点坐标(这次查找必然成功)，返回k，t。随后k、t乘二即为在b数组中坐标。此坐标将作为参数进行翻牌或者查询。

pushdown

每当翻牌或者查询需要向下递归时，均应该进行一次pushdown。左右子树的tag分别加上父节点的tag，而左右子树的sum也做相应的调整。

翻牌

已经通过二分搜索得到了查找的区间两端点，随后开始对全树进行递归搜索。约定如下参数：i当前树的编号，tree_le为当前搜索的树的左端，tree_ri为当前搜索树的右端，le为当前搜索的左端点，ri为当前搜索的右端点。分以下类别讨论：

  1. ri<tree_le，直接返回。

  2. tree_ri<le，直接返回。注意多个if语句加上查询操作和翻牌操作都是左闭右闭的，很容易容易出现情况重叠。首先确定判定的顺序，且尽量使用else语句。

  3. 树并不涵盖在区间内 ，然而和区间有交集。首先如果本级的tag!=0，则将本级的tag pushdown，随后计算mid，左右分别递归向下递归，左子树的编号为2*i，右子树的编号为2*i+1，然后通过mid给出左右子树的范围。(可以见得左右子树的范围虽然固定，但是并未存下来，而是每次通过计算所得，这样牺牲时间换了空间)。最后还需要及时将线段和反馈给父节点。

  4. 树完全涵盖在区间内。结束递归，sum[i]+=len，tag[i]+=1，tree_le和tree_ri是b数组中的坐标，讨论其奇偶性。按照int k = tree_le&1+tree_ri&1对len做如下讨论：

    • case 0，if(tree_le=tree_ri)则len=1。反之，if(tree_le=tree_ri)&&(tree_le&1==1)则len= a[(tree_le>>1)+1]-a[tree_le>>1]。如果if(tree_le!=tree_ri)则len=a[tree_ri>>1]-a[tree_le>>1]+1
    • case 1，len=a[(tree_ri+1)>>1]-a[tree_le>>1]
    • case 2，则不论tree_le==tree_ri，均有len=a[(tree_ri+1)>>1]-a[tree_le>>1]-1

    然而注意到以上接口的结果高度相似，实际上不用讨论奇偶性，利用位运算化简。

len接口

由于len接口出现较多，考虑单独写为一个接口。该接口的参数是b数组的坐标两端，实现基础为4，实际上可以写的较为简便。

查询

已经通过二分搜索得到了查找的区间两端点，随后开始对全树进行递归搜索。i当前树的编号，tree_le为当前搜索的树的左端，tree_ri为当前搜索树的右端，le为当前搜索的左端点，ri为当前搜索的右端点。分以下类别讨论：

  1. ri<tree_le，直接返回。
  2. tree_ri<le，直接返回。
  3. 树并不涵盖在区间内 ，然而和区间有交集。，首先如果本级的tag!=0，则将本级的tag pushdown，随后计算mid，左右分别递归向下递归，左子树的编号为2*i，右子树的编号为2*i+1，最后返回sum
  4. le<=tree_le<=tree_ri<=ri，结束递归，返回sum。

二、复杂度分析

空间复杂度

通过上述分析，实际上空间复杂度与n无关，而取决于m。首先我们使用了数组a存储初始的操作列表，随后复制一份a到了sign数组（因为a经过排序后已经打乱了顺序，不再能表示原本的操作）。然后我们对于假想的区间数组b开辟了线段树（通过sum、tag两个数组来模拟），考虑到线段树的大小在最坏情况下是为b数组的四倍，故而线段树加上起初存下的操作，排序后的操作，整体的空间复杂度仍然为为o(m)。进一步考虑最坏情况下，当m=200_000时，最大的空间占用不过：

时间复杂度

接下来考虑时间复杂度，首先读入所有数据后，进行排序去重等操作。拷贝数组复杂度为o(m)，排序的时间复杂度为o(mlog(m))，随后进行去重，复杂度为o(m)，这一步复杂度为o(mlog(m))。

每一步翻牌操作和每一步查询操作实际上渐进时间复杂度相同。由于操作当中仅仅只是将空间扩大了一倍，并没有提升线段树的渐进复杂度，故而线段树的复杂度仍然为o(mlog(m)）。

综上，总体的时间复杂度为o(mlog(m))。

去重排序
