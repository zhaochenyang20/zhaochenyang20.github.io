PA2-1 Build题解

一、算法分析

首先考虑如何模拟一棵多叉树的行为。参照讲义所讲，我们采取长子兄弟法来模拟这样的一棵树。每个节点需要储存的信息为直接前驱编号，直接后续编号，长子编号，幼子编号，父亲编号。并且规定倘若不存在某一关系节点，则编号为0。毕竟根节点的编号从1开始。

除关系节点之外，还需储存子树大小信息，子树高度信息，后缀节点最大高度。

如上6个信息采用unsigned int储存足矣。故而构造节点结构体，开辟足够大小空间即可。

接下来考虑改进递归实现的后序遍历算法来生成初始信息。

最初我考虑通过递归来实现这个功能：

标准的后续遍历为先遍历lc，随后遍历rc，最后visit节点本身。然而本题的后缀节点最大高度这一数据实际上需要依赖于某一节点所有的后续节点。故而需要从last_child反向遍历至first_child。这一操作实际上是反向的后序遍历。考虑以如下方式实现：

如果一个节点是叶子，则设计size=1,height=0,subffix=0，随后返回。

从幼子节点开始反向遍历，直到长子节点。

随后该节点的size等于所有自己子节点的size之和。subffix为长子节点和右邻居的最大值。height为长子节点的subffix+1。

然而实际上由于递归的空间开在函数栈上，而输入规模为1000_000。

如果一棵树完全退化为一维的线性数组，则很有可能因为不断递归进入幼子节点而爆栈。（如果有同学的代码用递归能过全集测试，我去试试能否用这个worst case卡掉他们），所以我们需要递归改为迭代。具体可以参见第三部分第一节。

接下来考虑动态操作。

功能设计

int readnode()

readnode()接口用于给出被表示的节点。起初我考虑通过一个vector将参数传入后再返回被表示的节点，然而并不需要如此操作，毕竟 vector在此处并不优化时间，反而浪费了空间。

考虑到实际上会读入非法路径，故而设计四个全局变量，用于实现int readnode()接口。

child_number表示读入的子节点的编号，length表示读入的路径长度，real_vertex和tmp_vertex分别表示在树上，沿着路径深入时，真实的节点和临时的节点。注意到因为路径可能非法，故而我们将tmp_vertex设置为0以表示路径已然非法。且在函数开始时，将tmp_vertex、real_vertex初始化为1，表示必然会走到树根节点。

随后，读入length，然后开启循环。循环内部每次读入child_number，并调用void search(child_number)来设置tmp_vertex、real_vertex。循环结束后返回real_vertex并用detach_vertex、attach_vertex接收即可。

void search(int child_number)

如同readnode中叙述，search是在模拟在树上的行走过程。首先，如果tmp_vertex已然等于0，也即下行路线到达了叶节点，则直接返回。反之，我们从real_vertex节点的长子节点开始不断移动即可。

void detach(int detach_vertex)

模拟将一个节点从父亲的树中摘除。首先对于其左右节点，让左右邻居的邻居接口直接相互指向即可。同时特判被摘除节点是否为父节点的长子节点或者幼子节点，如果是则应该将长子节点或幼子节点移交。

其后考虑先更新所有受到影响的节点的信息。具体而言，需要更新的是size、height、subffix。

对于size的更新，只需沿着父节点不断向上，减去当前节点的size。

对于height、subfix，从被摘除节点的parent开始，通过一个while循环，如果有左邻居则向左，如果没有左邻居则向上。在每次循环内，对于左邻居，后缀树最大高度subffix为左邻居的高度与左邻居（新的）右邻居的subffix中的最大值，而高度不会发生改变。如果左邻居的左邻居仍然存在，则继续向左。如果不存在，则已经到达长子。在长子的循环内，父节点的高度等于长子的subffix+1。随后转入父节点的循环。

void attach(int attach_number，int rank)

约定attach_number为待插入位置的父节点。

考虑如何获得实际待插入的位置。类似于search接口的方法，我们通过两个全局变量tmp_insert和real_insert配合rank的while循环获得被attach的位置当前的节点(注意需要将两个全局变量在每次使用前设置为-1)。特别的，倘若tmp_insert!=real_insert，则意味着被插入的位置当前节点为幼子节点。

接着我们应该获得将待插入节点与父节点的儿子链条接起来。如果插入到链条中部，则将原来此处节点的左邻居的右邻居指针指向待插入节点，原来此处节点的左邻居指针指向待插入节点。如果是在长子或者幼子的位置，则注意更新父节点的长子或者幼子指针。

类似于detach，我们需要更新受影响的节点的信息。

对于size的更新，只需沿着父节点不断向上，加上当前节点的size。

对于height和subffix，被插入节点的height不变，而subffix为高度与右邻居的subffix的最大值。而后，从被插入节点的左邻居开始，在每次循环内，对于左邻居，后缀树最大高度subffix为左邻居的高度与左邻居（新的）右邻居的subffix中的最大值，而高度不会发生改变。如果左邻居的左邻居仍然存在，则继续向左。如果不存在，则已经到达长子。在长子的循环内，父节点的高度等于长子的subffix+1。随后转入父节点的循环。

void query(query_number)

直接返回维护好的信息即可。

更新信息

由于摘除和添加节点都会对原来的树造成影响，需要更新全树。故而单独将此写为一个接口。且做语义约定：摘除时，在detach函数中将原来的树与被摘除子树完全脱离。被摘除节点的左右邻居的左右邻居指针应该互相指向（这里需要特判左右邻居是否为空，也就是0，因为尽量不应该更改点0的属性，也即不可让0节点的某一个邻居指针指向非0的节点），如果被摘除节点的左右邻居有节点0，则不应该修改左右邻居的邻居指针，且此时也即被摘除点是原本父节点的长子或者幼子，也应该将长子指针或者幼子指针更新。

以及，因为更新时，对于size的更新方式并不统一，故而将size的更新放置在detach、attach中，只需沿着parent指针做一while循环，不断加上或者减去size即可。

这样约定后，在摘除过程中，我们并没有改变被摘下节点的现在的指针位置，但是原本的树中已经完全没有了次节点。如此以来，对摘除过程进行更新。

注意到实际上受到影响的点是被摘除节点沿着向左向上的方向直到到达根节点的路径上的全部节点。

其次考虑更新subffix、height。先更新subffix再更新height。

做一while循环，沿着先向左的方式更新subffix，在向左的过程中subbfix=max{height,新的右邻居的subffix}。当到达最左侧后，试图向上。如果向上成功，则height=长子的subffix+1，且再次向左。这一过程即更新了所有受影响节点的height与subffix。

完成了全树的更新后，为了确保运行安全，将被摘除节点的parent、left_sibling、right_sibling指针清零以便于之后再使用。

对于插入的语义预定，我们在调用update之前将接口全部接上，也即如果被插入的位置现在有节点，则插入节点的right_sibling指向被插入位置原来的节点，如果被插入位置现在的节点的left_sibling不为空，则被插入位置在儿子链条的中部，相互回扣上链条即可。如果被插入位置现在的节点的left_sibling为空，则新插入节点为父节点的长子，需要回扣上相应的指针。如果被插入位置现在没有节点，则待插入节点为幼子，更改父节点的幼子指针，且判断此时待插入节点的父亲的长子是否为空。如果长子为空，则修改父亲的长子指针。如果长子不为空，则待插入位置原本的节点的left_sibling的右邻居指针和待插入节点进行回扣。

随后调用更新接口即可。

二、复杂度分析

时间复杂度

构建树的拓扑结构的过程仅仅与节点数目有关系，故而时间复杂度为o(n)。

利用辅助栈实现信息初始化时，每一个节点必然且仅仅入栈、出栈各一次，故而整体的时间复杂度为0(n)。

子树移动过程，移动本身只设计了寻找节点的时间消耗，故而时间复杂度为o(cost)。

移动过程中维护大小时，只与节点深度有关，故而复杂度仍然是o(cost)。

移动过程中维护高度和subffix，实际上是逆向的查找过程，花费也是o(cost)。

综上所述，总体的时间复杂度为o(cost+n)，根据数据范围可知符合要求。

空间复杂度

每个节点，记录了大小，长子，幼子，左邻居，右邻居，高度，后缀，总体的空间消耗为o(n)。

为了利用辅助站完成迭代操作，开设的辅助栈也为o(n)。

为了完成对于辅助栈的标记，开设了bitmap，空间消耗也为o(n)。

综上所述，空间复杂度为o(n)。

三、相关问题

在这次作业过程中，集中解决了如下问题。

递归改迭代

在最开始我考虑子树信息的初始化时，采用了递归版本实现，但是正如前文所述，递归可能会被完全线性的序列卡掉，造成栈溢出。

故而参考了邓老师上课所讲的后续迭代算法，并与我的朋友刘明道、高焕昂讨论后，我利用辅助栈结合标记的方法将递归改为了迭代。

其中辅助栈利用数组模拟即可。

而标记数组，为了节省空间，这里利用bitmap来完成。
