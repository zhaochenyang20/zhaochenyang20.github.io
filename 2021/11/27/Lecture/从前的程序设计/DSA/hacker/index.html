PA2-2 Hacker题解

题意解析：

题目给出，对于任意一个字符串，我们有： \[ \text{for }\forall d_{ki} \in D_k\\ 密文：\text{int }M_{ki}=Hash(d_{ki})=CRC32(d_{ki}+salt)\\ \] 取质数10000019为T,记录数据结构Table[10000019]为所需的哈希表，随后以线性试探法来处理哈希冲突。table为一个pair数组，每个pair储存\((M_{ki},d_{ki})\)键值对，而\((M_{ki},d_{ki})\)储存在table[M_{ki}%T]的位置上。

接口设计：

set

首先对于每个pair设计set函数，set的参数为unsigned char*a,int length。首先检测该位置是否已经有节点，如果没有则将pair存入该位置，反之已经产生了哈希冲突。不断向后++直到出现空，插入即可。

search

每当传入一个M值，则计算出M%T，随后在table的相应位置开始查询。

对于每一个节点，倘若查询到其节点的pair值等于M，则意味着为查询成功。随后一直遍历到空，倘若这一过程中还发生了查询成功，则意味着dupulicate，否则查询成功，输出查询成功的字符串，同时将字符串的首字母存入队列中。在队列中进行一次set。

队列与更新

考虑到题目中的更新需求，实际上利用一个队列结构来实现即可。每次成功输出时，将输出的首字母入队——这意味着，其实我们只需要将队列开为一个100_0010的char数组即可。长度超过8以后，每次将head++,tail++。而后从tail读取出长度为6、7、8的字符串，调用set即可。

递归初始化

设计一个利用辅助栈的DFS算法，称为init(int depth)。在每一层中设置stack[depth]=each_in_list，而后set然后递归即可。

二、复杂度分析

时间复杂度

枚举部分虽然采用了递归方式，但是其时间复杂度必然与迭代相同。故而复杂度为： \[ O\left(\sum\limits_{i = 1}^5 18^i \right) = O(18^5)=O(1)(虽然常系数很大) \] 每次询问会进行O(1)次散列的查询，随后可能会插入，而插入的复杂度为O(1)，因此平均插入时间与查找时间均为O(1)。窃听的次数为n，总体的时间复杂度为O(n)，综上所述，时间复杂度为o(n)。

空间复杂度

起初五位的字符共有1889568，随后进行n次查询，每次查询最大会插入3个新的字符串，故而空间复杂度为O(n)。

三、相关问题

在这次作业过程中，集中解决了如下问题。

输入格式的处理

第一次编写程序时，我利用scanf处理字符串，在本地调试通过后上传至oj却始终无法通过。

随后改为了cin读取，成功通过了程序。

请教助教后，应该是我关闭了sync_with_stdio后，同时采用了cin与scanf，导致了发生未定义行为。

去重

注意到查找范围是个集合，集合内没有重复元素。（之所以会duplicate是因为两个不同的明文对应了同一个暗纹，而非出现了相同明文）

因此在查找过程中的插入需要进行去重。具体而言，如果除余法得到的位置有元素，且当前元素的str不等于待插入的元素，则向后查询，否则直接退出。当查询到空位置时，插入即可。

控制空间

无论是设计为class还是struct，由于内存的对齐机制，都会浪费一部分空间。故而将class直接用数组模拟，节省了部分空间。
